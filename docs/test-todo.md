# Test consolidation plan: `windows-bindgen`, `windows-rdl`, `windows-metadata`

## 1. Background — current testing strategies

The repo currently has five overlapping testing strategies for the parser/generator stack:

| # | Location | What it does | Strengths | Drawbacks |
|---|---|---|---|---|
| A | `crates/tests/roundtrip/rdl` | Auto-scans `src/*.rdl`, reads RDL → winmd → RDL, asserts the file is byte-stable. ~60 cases, build.rs generates `#[test]` functions. | Concise, catches a huge class of reader/writer asymmetries, easy to add new cases. | Only proves *fixed-point*; symmetric bugs in both reader & writer slip through. Doesn't validate the *intermediate* winmd or any generated Rust. Hard-codes the `Test` namespace filter. |
| B | `crates/tests/roundtrip/clang` | Auto-scans `src/*.h`, runs Clang → RDL → winmd → RDL. ~35 cases. | Same. Also exercises clang frontend. | Same as (A); also `target_pointer_width = "64"` only. |
| C | `crates/tests/roundtrip/bindgen` | Auto-scans `src/*.rdl`; reads RDL → winmd → RDL (roundtrip) **and** runs `windows-bindgen` to commit `src/*.rs`. | Adds bindgen output validation in addition to the RDL roundtrip. | Generated `.rs` files are committed and reviewed via diff. Only 7 cases today; the same coverage idea isn't applied to the older `tests/libs/bindgen` golden snapshots. |
| D | `crates/tests/libs/rdl/tests/*.rs` (≈25 files) | Mix of: bespoke roundtrip (`use-declarations`, `directory`, `split`, `nested-arches`, `path`, `struct_fields`, `struct_values`, `mod-recursive*`, `default-interface`, `const-underlying*`, `guid-derive`, `fn_abi`, `nested-packing`, `exclusive-to`, `assembly_name`); error/panic harnesses (`panic.rs` ~50 inline RDL strings, `error.rs`, `writer_errors.rs`, `invalid_output.rs`); flag/feature checks (`opt.rs`, `filter.rs`). | Captures behaviour the roundtrip framework can't (errors, type-system invariants, file-IO modes, golden Rust output). | Each test reinvents temp-file plumbing, every "happy-path" file duplicates what roundtrip tests already do. ~half of these could be deleted by porting the `.rdl` file into `roundtrip/rdl/src/`. |
| E | `crates/tests/libs/bindgen/src/*.rs` (~110 files) generated by `crates/tools/bindgen` | Imperative tool runs `bindgen` with a one-liner per case against the prebuilt `default/*.winmd`, commits the output. A few `tests/*.rs` exercise the resulting bindings at runtime. | Concise definition (one line per fixture). Drives diff-based review. | The list of cases lives in tool source, not as data; cannot be filtered/parallelized; no equivalent for *RDL → bindgen* output (which is what most users actually drive). |
| F | `crates/tests/libs/metadata/tests/*.rs` | Direct `writer::File` API tests (`empty`, `struct`, `attribute`, `class`, `interface`), `reader` queries against `default/Windows*.winmd`, `merge.rs` (RDL→winmd, then `windows_metadata::merge()`, then RDL writer). | Tests the metadata APIs in isolation, the only place merge is exercised. | `merge.rs` could be reframed as a roundtrip case (RDL+RDL → merged winmd → golden RDL). Direct-writer tests are valuable but don't compose with the rest. |
| G | `crates/tests/misc/**`, `crates/tests/winrt/**` | Older runtime tests for compiled bindings against real `windows`/`windows-sys` crates. | Only place exercising actual Windows behaviour. | Out of scope for *generation* testing; many duplicate what bindgen-output snapshots can prove statically. Several are obsolete vs. the newer `tests/libs/bindgen` framework. |

## 2. Key drawbacks of the current roundtrip framework we must keep addressing

Roundtrip is convenient but has three real limitations that drive the existence of the other test styles:

1. **Fixed-point ≠ correctness.** A reader/writer pair that mis-encodes the same construct will still roundtrip. We need *cross-checks* (golden Rust from bindgen, structural assertions on the winmd, ECMA-335 `ildasm`-style dumps, runtime tests).
2. **No assertion on intermediate winmd.** Tests can't easily say "this RDL produces a `TypeRef` to `System.Enum` with these flags."
3. **Hard to express *negative* cases.** Errors, panics, conflicting attributes, invalid types — all live outside the roundtrip framework today.
4. **Limited input shapes.** Roundtrip uses one filter (`Test`) and one entry point (`reader().input(rdl)`); many code paths (CLI argv, multi-input directories, `--reference`, `--split`, `--no-deps`, `--flat`, `--sys`) need separate fixtures.

## 3. Goals

1. **One canonical "fixture-per-feature" model** — each language feature is described by a small directory of inputs and one or more golden outputs; tests are auto-generated from the directory, not enumerated in code.
2. **Layered assertions** — every fixture can opt into roundtrip, golden-winmd, golden-Rust, and (where applicable) compile/run checks, instead of forcing each test author to wire it up by hand.
3. **Centralize negative tests** as data, not as `should_panic` boilerplate.
4. **Eliminate duplication** between `tests/libs/rdl`, `tests/libs/metadata`, `tests/libs/bindgen`, and `tests/roundtrip/*`.
5. **Increase coverage** in the gaps revealed by the audit (error sites, merge, multi-arch, references, CLI, split mode, attribute encoding edge cases, large real-world winmds).

## 4. Proposed unified architecture

### 4.1 A single fixture format under `crates/tests/fixtures/`

Each fixture is one directory:

```
crates/tests/fixtures/<group>/<name>/
    input.rdl              (or input.h for clang fixtures, or input.toml for CLI)
    refs/                  (optional extra inputs / referenced winmds, by filename)
    expected.rdl           (writer-roundtrip golden)
    expected.winmd.txt     (optional: human-readable disassembly of the winmd)
    expected.rs            (optional: bindgen golden Rust)
    expected.err           (optional: expected reader/writer error message)
    fixture.toml           (declares which checks to run, filter, flags, --sys etc.)
```

`fixture.toml` is the single source of declarative knobs that today are hard-coded into per-test Rust files (filter name, `--sys`, `--no-deps`, `--flat`, `--reference`, `--split`, multi-arch bits, `target_pointer_width`, optional `expected_err`).

### 4.2 A single `test_fixtures` harness crate

A new crate (replacing today's three separate `roundtrip/*` crates) auto-discovers every fixture directory at build time and emits one `#[test]` per directory per enabled check:

- `roundtrip_<name>` – read inputs → write winmd → write back RDL → diff against `expected.rdl`
- `winmd_<name>` – dump `expected.winmd.txt` and diff (uses `windows_metadata::reader`)
- `bindgen_<name>` – run `windows_bindgen` with declared options and diff `expected.rs` (and `cargo build` it via a generated `lib.rs`)
- `error_<name>` – assert reader/writer fails with `expected.err` (replaces all `should_panic` boilerplate; far more diffable than embedded strings)
- `clang_<name>` – when an `input.h` is present, run Clang → RDL → diff
- `merge_<name>` – when multiple inputs are declared, run `windows_metadata::merge()` and diff
- `cli_<name>` – when `argv.txt` is present, drive `bindgen()`/`bindgen-cli` end-to-end

Update mode: `UPDATE_GOLDEN=1 cargo test` rewrites `expected.*` files from actual output, so authors add a fixture by dropping a directory and running tests once with the env var. (Same UX `tool_bindgen` provides today, but for *all* outputs uniformly.)

### 4.3 Replace `tool_bindgen` with the same harness

Today `crates/tools/bindgen` enumerates ~110 imperative `test()` calls. Convert each call to a fixture directory whose `fixture.toml` records the CLI args, and let the harness regenerate `expected.rs`. The tool becomes a 5-line wrapper that simply runs the harness in update mode. This collapses three regeneration mechanisms into one.

### 4.4 Extend the same model to clang fixtures

Replace `crates/tests/roundtrip/clang` with fixtures whose `input.h` triggers the `clang_*` check. This restores cross-platform skipping via `fixture.toml` (`requires_target_pointer_width = 64`) instead of a top-of-file `#![cfg]`.

### 4.5 Negative-case fixtures replace `panic.rs`

Each `should_panic` test in `crates/tests/libs/rdl/tests/panic.rs` (~50 cases) becomes a fixture with `input.rdl` and `expected.err` containing the canonical error message. Benefits: easier diff review, no escaped string literals, easy to grep, the same harness runs them.

### 4.6 Keep direct-API tests, but consolidate

`crates/tests/libs/metadata/tests/{empty,struct,attribute,class,interface,reader}.rs` exercise the `windows_metadata::writer::File` builder API directly. These don't fit the fixture model and should stay, but should move under `crates/tests/libs/metadata/tests/` as the *only* `tests/libs/metadata` content, with `merge.rs`, `merge_*.rdl`, `assembly_name.rs`, `load_library.rs` migrated into fixtures.

## 5. Concrete consolidation list (what to delete/rewrite)

A. **Migrate to fixtures (delete the bespoke `.rs` test):**
- `tests/libs/rdl/tests/struct_fields.rs`, `struct_values.rs`, `nested-arches.rs`, `nested-packing.rs`, `mod-recursive.rs`, `path.rdl/.rs`, `default-interface.rs`, `const-underlying*.rs`, `guid-derive.rs`, `fn_abi.rs`, `directory.rs`, `split.rs`, `use-declarations*.rs`, `exclusive-to.rs`, `opt.rs`, `filter.rs`, `assembly_name.rs`, `invalid_output.rs`.
- `tests/libs/metadata/tests/merge.rs` + all `merge_*.rdl` (use the `merge_*` check).
- All seven cases in `tests/roundtrip/bindgen/src/*.rdl` (now live under fixtures with `bindgen_*` enabled).
- All cases in `tests/roundtrip/rdl/src/*.rdl` and `tests/roundtrip/clang/src/*.{h,rdl}`.

B. **Convert to error-fixtures:**
- All ~50 `should_panic` tests in `tests/libs/rdl/tests/panic.rs`.
- The arch error tests in `tests/libs/rdl/tests/error.rs`.
- The two writer-error tests in `tests/libs/rdl/tests/writer_errors.rs`.

C. **Convert to bindgen-fixtures (replaces `tool_bindgen` enumeration):**
- All entries in `crates/tools/bindgen/src/main.rs`.
- The committed `crates/tests/libs/bindgen/src/*.rs` files become `expected.rs` golden files inside their fixtures.
- The runtime `tests/libs/bindgen/tests/{bool,deps,delegate_*,panic,ref_params}.rs` stay (they exercise generated bindings on Windows) but consume the fixtures via path.

D. **Keep as-is:**
- `tests/libs/metadata/tests/{empty,struct,attribute,class,interface,reader}.rs` (direct builder/reader-API unit tests).
- `tests/libs/rdl/tests/error.rs::error_display` (formatting unit test).
- All `crates/tests/misc/**` and `crates/tests/winrt/**` runtime suites — these test the published `windows` / `windows-sys` crates, not the parser/generator stack.

## 6. New coverage to add (gaps surfaced by the audit)

1. **Bindgen-from-RDL fixtures.** Today only roundtrip→bindgen exists for 7 RDL inputs and only bindgen-from-default-winmd exists for ~110 cases. Cross the matrix: every `tests/libs/bindgen` case should additionally have an RDL-source variant where feasible, exercising the full RDL→winmd→bindgen pipeline.
2. **Multi-arch fixtures.** Only `merge_arch_*` exercises `arch_input`; add fixtures combining x86+x64+Arm64, with nested types and conditional members, and golden `SupportedArchitecture` bitmasks.
3. **Reference-winmd tests.** Add fixtures that import a referenced winmd and assert that *no* `TypeDef` for the referenced type is emitted (only `TypeRef`). Today this is covered indirectly by `use-declarations-ref` only.
4. **CLI / argv fixtures.** Drive `windows_bindgen::bindgen` with full argv, including malformed flags (negative cases), unknown filters, missing `--in`, conflicting `--reference` values.
5. **Split-mode and directory-input fixtures** with multiple namespaces, asserting per-file output.
6. **Merger negative cases.** Conflicting type definitions across inputs, mismatched arches, missing `arch_input` bits — none currently tested.
7. **Attribute encoding edge cases.** Empty arg lists, named-only args, enum-typed args resolved across input boundaries (the existing `writer_errors_on_missing_enum_type` is the only such test).
8. **Determinism.** Re-run each bindgen fixture twice, diff the two outputs (catches HashMap-iteration bugs that have bitten the writer historically; today only `sort.rs` exercises ordering implicitly).
9. **`assembly_name` derivation** for file vs. directory output, with and without explicit `--name`.
10. **Roundtrip stability for `default/Windows*.winmd`.** Add a fixture that takes the prebuilt large winmds, writes them out as RDL, reads them back, and diffs the winmd bytes (catches regressions in real-world coverage rather than synthetic mini-cases).

## 7. Phased rollout

1. **Land the harness and fixture format** alongside the existing tests; demonstrate it on 3–5 fixtures from each category.
2. **Migrate `tool_bindgen`** to the harness; delete `crates/tools/bindgen/src/main.rs` body.
3. **Migrate panic/error tests** (mechanical translation; can be scripted).
4. **Migrate bespoke `tests/libs/rdl/tests/*` files**, deleting each as its fixture lands.
5. **Collapse `tests/roundtrip/{rdl,clang,bindgen}` crates** into the harness; delete them.
6. **Add the new coverage** from §6 once the harness is stable.
7. **Document** the fixture format in a `crates/tests/fixtures/README.md` so contributors add data, not Rust.

## 8. Outcome

- ~200 hand-rolled test files / 180 imperative `test()` calls collapse to a single harness + ~250 fixture directories.
- One way to add a test (drop a directory), one way to update goldens (`UPDATE_GOLDEN=1`), one way to read CI failures (file diffs, not panic strings).
- Coverage strictly increases: every fixture gets a winmd-structure check and a determinism check for free; gaps in §6 close; the older one-off `.rs` tests stop drifting from the roundtrip suite.
- Roundtrip semantics remain (so we keep their convenience) but are no longer the *only* signal — golden Rust + structural winmd + error-string fixtures jointly close the "symmetric reader/writer bug" hole.

# windows-reactor — internals

> Maintenance notes for `windows-reactor`. For usage, see the [guide](../guide/windows-reactor.md).

- 📁 [Source](https://github.com/microsoft/windows-rs/tree/master/crates/libs/reactor)

## How it's built

The WinUI bindings (`src/bindings.rs`) and the control/property/event codegen
(`src/generated.rs`, `backend/winui/generated_set_prop.rs`, and
`generated_attach_event.rs`) are generated by `tool_reactor` from
`crates/tools/reactor/src/winui.toml` plus the WinUI `.winmd` metadata. The
hooks runtime, element tree, and diffing engine are hand-written. Regenerate
with `cargo run -p tool_reactor`, then verify with `cargo check -p windows-reactor`.

## Testing

Unit tests live in `test_reactor` (headless). Integration tests live in
`test_reactor_selftest`, which launches a real WinUI window — pass `--headless`
for CI.

# windows-threading

> Simple, safe access to the Windows thread pool.

- 📦 [crates.io](https://crates.io/crates/windows-threading)
- 📖 [API reference (docs.rs)](https://docs.rs/windows-threading)
- 🚀 [Getting started](../../crates/libs/threading/readme.md)
- 📁 [Source](https://github.com/microsoft/windows-rs/tree/master/crates/libs/threading)

`windows-threading` exposes the Windows thread pool through a small, safe API.
`submit` runs a closure on a pool thread, `for_each` runs a closure over an
iterator in parallel and waits for completion, and the `Pool` type offers direct
control including thread limits and scoped submission.

## How it's built

`src/bindings.rs` is generated by `tool_bindings` from
`crates/tools/bindings/src/threading.txt`; `Pool` and the `submit`/`for_each`
helpers are hand-written.

## Testing

Run `cargo test -p windows-threading`; see also the workspace test crates.

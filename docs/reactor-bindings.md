# Bindings

The file `crates/libs/reactor/src/bindings.rs` is auto-generated by
`windows-bindgen` from WinUI 3 metadata. **Never edit it by hand.**

## Regenerating

```sh
cargo run -p tool_bindings    # regenerate all bindings (reactor + test)
cargo build -p windows-reactor    # verify reactor compiles
cargo build --workspace           # verify everything including tests
```

## Minimal filter syntax

The filter lives in `crates/tools/bindings/src/reactor.txt` (and
`reactor_test.txt` for test-only types). It uses the `--minimal` mode which
is opt-in: you list exactly the methods you need, and bindgen computes the
minimal type closure.

### Entry forms

| Syntax | Effect |
|--------|--------|
| `Ns.IFace::{m1, m2}` | Specific interface methods (preferred) |
| `Ns.IFace::*` | All methods on an interface (only for `--implement` interfaces) |
| `Ns.Class::{CreateInstance}` | Class instantiation only (no instance methods) |
| `Ns.Type` | Bare type (enum, struct, delegate, constant) |

### Raw metadata names

Methods must use their raw names from metadata:

- Property getter: `get_PropertyName`
- Property setter: `put_PropertyName`
- Event subscribe: `add_EventName`
- Event unsubscribe: `remove_EventName`
- Regular method: `MethodName` (or its overload name, e.g. `SetPresenterByKind`)

### Class instantiation with `{CreateInstance}`

Most WinUI controls use `{CreateInstance}` â€” this includes the class type with
its `new()` method but does NOT expand any instance interfaces. This keeps the
output minimal. Instance methods come from explicit interface entries instead.

```
// Class entry (instantiation only):
Microsoft.UI.Xaml.Controls.TextBlock::{CreateInstance}

// Explicit interface entries for the methods actually used:
Microsoft.UI.Xaml.Controls.ITextBlock::{put_Text, put_TextWrapping, put_IsTextSelectionEnabled}
```

For composable classes, `{CreateInstance}` expands the composable factory interface.
For default-activatable classes, it just includes the class (new() is generated
from the `has_default_constructor` attribute).

### `--implement` interfaces

Interfaces listed in `--implement` keep ALL methods (for implementing traits).
These are the only entries that should use `::*`.

### Interface vs class methods

Instance methods live on their owning interface. Access via `widget.cast::<IFoo>()?.put_Bar(v)`,
or call directly if it's on the class's default interface (via `Deref`).

## Pruning workflow

To find which methods a filter entry actually needs:

1. Comment out or remove the entry from `reactor.txt`
2. `cargo run -p tool_bindings && cargo build -p windows-reactor`
3. Compiler errors show exactly which methods are missing
4. Map Rust names â†’ raw names: `SetX` â†’ `put_X`, `X()` â†’ `get_X`, etc.
5. Add the interface entry with explicit methods: `Ns.IFoo::{put_X, get_Y}`
6. Regenerate and verify

### Example

```
// Before (class with all methods):
Microsoft.UI.Xaml.Controls.InfoBadge::*

// After (class for instantiation + only the interface methods used):
Microsoft.UI.Xaml.Controls.InfoBadge::{CreateInstance}
Microsoft.UI.Xaml.Controls.IInfoBadge::{put_Value}
```

## Architecture notes

- **Classes don't flatten instance methods** in minimal mode â€” each interface
  has its own `impl` block. Call via `widget.cast::<IFoo>()?.put_Bar(v)`.
- **Static/factory methods** ARE on the class (for factory caching).
- **`Deref` to default interface** â€” classes deref to their default interface,
  so default interface methods can be called directly without casting.
- **Event-args types look unused** but are required by generated closure
  signatures. Don't remove them.
- **`--implement` types** keep all methods unconditionally.
- **Enum constants** (e.g. `NavigationViewBackButtonVisible::Visible`) cannot be
  pruned individually â€” including the type includes all variants.

## COM (Win32) interfaces in minimal mode

The `--minimal` filter also works for Win32 COM interfaces (e.g. DXGI, D2D,
DWrite). The same `Ns.IFace::{Method}` syntax applies:

```
Windows.Win32.Graphics.Dxgi.IDXGIFactory2::{CreateSwapChainForComposition, CreateSwapChainForHwnd}
Windows.Win32.Graphics.Direct2D.ID2D1RenderTarget::{BeginDraw, EndDraw, Clear, DrawLine}
```

When a COM method is explicitly listed in the filter, bindgen emits the full
method body and vtable function pointer. Methods NOT in the filter become
`usize` slots (preserving vtable layout without generating code for them).

The type closure is computed automatically: parameter and return types of
requested methods are recursively pulled in (structs, enums, other interfaces
needed for casts). You do NOT need to explicitly list dependency types â€” just
list the methods you call.

**Example** (`crates/tools/bindings/src/canvas.txt`):
```
--out crates/libs/canvas/src/bindings.rs
--flat --minimal

--filter
    Windows.Win32.Graphics.Dxgi.IDXGIFactory2::{CreateSwapChainForComposition, CreateSwapChainForHwnd}
    Windows.Win32.Graphics.Dxgi.IDXGISwapChain::{Present, GetBuffer, ResizeBuffers}
    ...
```

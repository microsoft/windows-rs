// Filter for windows-composition's bindings (Windows.UI.Composition, the SYSTEM
// stack). tool_composition generates two modules from this single filter:
//
//   * bindings.rs        — the system stack, verbatim from this filter.
//   * bindings_lifted.rs — the lifted Microsoft.UI.Composition stack, derived by
//                          dropping every `// region: system-only` block and
//                          rewriting `Windows.UI.Composition.` to
//                          `Microsoft.UI.Composition.`.
//
// One filter is the single source of truth for both stacks; the wrapper source
// (visual.rs, shape.rs, ...) is compiled against whichever module the crate's
// features select (see docs/crates/windows-composition.md). Prune or extend the
// safe surface by editing this file, then re-run cargo run -p tool_composition.
//
// The system stack — unlike lifted Microsoft.UI.Composition — can host a visual
// tree directly in an HWND via ICompositorDesktopInterop::CreateDesktopWindowTarget,
// with no WinUI / Windows App SDK dependency. Those standalone-hosting entries are
// marked `// region: system-only` and excluded from the lifted surface.
